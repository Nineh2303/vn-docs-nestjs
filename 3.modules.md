# Chương 3. Modules

## 3.1 Mục tiêu chương học

Sau khi hoàn thành chương này, bạn sẽ hiểu:

- Module là gì trong NestJS.
- Vai trò của Module trong kiến trúc ứng dụng.
- Cách tổ chức Controller và Provider bằng Module.
- Cách chia nhỏ ứng dụng thành nhiều Module.
- Cách chia sẻ Provider giữa các Module.
- Cách sử dụng Imports và Exports.
- Cách xây dựng kiến trúc Module cho ứng dụng thực tế.

---

## 3.2 Module là gì?

Module là đơn vị tổ chức cơ bản trong NestJS.

Mọi ứng dụng NestJS đều phải có ít nhất một Module.

Thông thường đó là:

```typescript
AppModule
```

Ví dụ:

```typescript
import { Module } from '@nestjs/common';

@Module({})
export class AppModule {}
```

NestJS sử dụng Module để:

- Nhóm các Controller.
- Nhóm các Provider.
- Quản lý Dependency Injection.
- Xác định phạm vi truy cập của Provider.

Có thể hình dung:

```text
Module
 ├── Controllers
 ├── Providers
 ├── Imports
 └── Exports
```

---

## 3.3 Decorator @Module()

Một Module được khai báo bằng:

```typescript
@Module()
```

Ví dụ:

```typescript
@Module({
  controllers: [],
  providers: []
})
export class UsersModule {}
```

Decorator này cung cấp Metadata để NestJS xây dựng Dependency Graph cho toàn bộ ứng dụng.

---

## 3.4 Cấu trúc của Module

Một Module thường bao gồm:

```typescript
@Module({
  imports: [],
  controllers: [],
  providers: [],
  exports: []
})
export class UsersModule {}
```

Ý nghĩa:

| Thuộc tính | Mục đích |
|------------|----------|
| imports | Import Module khác |
| controllers | Khai báo Controller |
| providers | Khai báo Provider |
| exports | Cho phép Module khác sử dụng Provider |

---

## 3.5 Root Module

Mọi ứng dụng NestJS đều có Root Module.

Thông thường:

```typescript
@Module({
  imports: []
})
export class AppModule {}
```

Khởi động ứng dụng:

```typescript
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(3000);
}
```

NestJS bắt đầu xây dựng toàn bộ ứng dụng từ Root Module.

```text
AppModule
    ↓
UsersModule
    ↓
ProductsModule
    ↓
OrdersModule
```

---

## 3.6 Feature Module

Feature Module đại diện cho một domain hoặc tính năng cụ thể.

Ví dụ:

```text
src/
├── users/
├── products/
├── orders/
└── payments/
```

Mỗi thư mục có Module riêng:

```typescript
@Module({})
export class UsersModule {}
```

```typescript
@Module({})
export class ProductsModule {}
```

```typescript
@Module({})
export class OrdersModule {}
```

Điều này giúp:

- Dễ quản lý.
- Dễ mở rộng.
- Dễ bảo trì.

---

## 3.7 Đăng ký Controller trong Module

Ví dụ:

```typescript
@Controller('users')
export class UsersController {}
```

Đăng ký:

```typescript
@Module({
  controllers: [UsersController]
})
export class UsersModule {}
```

Nếu không khai báo:

```typescript
controllers: []
```

NestJS sẽ không thể tìm thấy Controller.

---

## 3.8 Đăng ký Provider trong Module

Ví dụ:

```typescript
@Injectable()
export class UsersService {}
```

Đăng ký:

```typescript
@Module({
  providers: [UsersService]
})
export class UsersModule {}
```

Provider sẽ được thêm vào Dependency Injection Container.

---

## 3.9 Module Context

Provider chỉ tồn tại trong Module đã khai báo.

Ví dụ:

```typescript
@Module({
  providers: [UsersService]
})
export class UsersModule {}
```

Provider:

```typescript
UsersService
```

chỉ khả dụng bên trong:

```typescript
UsersModule
```

Module khác sẽ không thể inject nếu chưa export.

---

## 3.10 Export Provider

Ví dụ:

```typescript
@Module({
  providers: [UsersService],
  exports: [UsersService]
})
export class UsersModule {}
```

Sau khi export:

```typescript
UsersService
```

có thể được sử dụng ở Module khác.

---

## 3.11 Import Module

Ví dụ:

```typescript
@Module({
  imports: [UsersModule]
})
export class OrdersModule {}
```

NestJS sẽ cho phép:

```typescript
OrdersModule
```

sử dụng các Provider đã export từ:

```typescript
UsersModule
```

---

## 3.12 Chia sẻ Provider giữa Module

Users Module:

```typescript
@Module({
  providers: [UsersService],
  exports: [UsersService]
})
export class UsersModule {}
```

Orders Module:

```typescript
@Module({
  imports: [UsersModule]
})
export class OrdersModule {}
```

Provider:

```typescript
UsersService
```

có thể được inject:

```typescript
constructor(
  private readonly usersService: UsersService
) {}
```

---

## 3.13 Shared Module

Một số Provider thường được dùng ở nhiều nơi.

Ví dụ:

```text
LoggerService
ConfigService
CacheService
```

Ta có thể tạo:

```typescript
@Module({
  providers: [
    LoggerService,
    ConfigService
  ],
  exports: [
    LoggerService,
    ConfigService
  ]
})
export class SharedModule {}
```

Các Module khác chỉ cần:

```typescript
imports: [SharedModule]
```

---

## 3.14 Global Module

Một Module có thể được đánh dấu là Global.

```typescript
import { Global, Module } from '@nestjs/common';

@Global()
@Module({
  providers: [ConfigService],
  exports: [ConfigService]
})
export class ConfigModule {}
```

Sau khi đăng ký:

```typescript
imports: [ConfigModule]
```

ở Root Module, Provider có thể được inject ở mọi nơi.

Không cần:

```typescript
imports: [ConfigModule]
```

trong từng Feature Module.

---

## 3.15 Dynamic Module

Một số Module cần cấu hình khi khởi tạo.

Ví dụ:

```typescript
ConfigModule.forRoot()
```

hoặc:

```typescript
JwtModule.register({
  secret: 'secret'
})
```

Các Module như vậy được gọi là Dynamic Module.

Ví dụ:

```typescript
@Module({})
export class DatabaseModule {

  static register() {
    return {
      module: DatabaseModule,
      providers: []
    };
  }

}
```

---

## 3.16 Module Re-export

Module có thể export lại Module khác.

Ví dụ:

```typescript
@Module({
  imports: [UsersModule],
  exports: [UsersModule]
})
export class SharedModule {}
```

Module khác chỉ cần:

```typescript
imports: [SharedModule]
```

---

## 3.17 Circular Dependency giữa Modules

Ví dụ:

```text
UsersModule
    ↓
OrdersModule

OrdersModule
    ↓
UsersModule
```

Điều này tạo ra Circular Dependency.

NestJS hỗ trợ:

```typescript
forwardRef(() => UsersModule)
```

Ví dụ:

```typescript
imports: [
  forwardRef(() => UsersModule)
]
```

Tuy nhiên nên hạn chế sử dụng.

Circular Dependency thường là dấu hiệu kiến trúc chưa tốt.

## 3.18 Module Organization

Một cấu trúc phổ biến:

```text
src/
├── users/
│   ├── users.module.ts
│   ├── users.controller.ts
│   └── users.service.ts
│
├── products/
│   ├── products.module.ts
│   ├── products.controller.ts
│   └── products.service.ts
│
├── orders/
│   ├── orders.module.ts
│   ├── orders.controller.ts
│   └── orders.service.ts
│
└── app.module.ts
```

Mỗi Domain có Module riêng.

## 3.19 Best Practices

### Một Module cho một Domain

Tốt:

```text
UsersModule
ProductsModule
OrdersModule
```

Không nên:

```text
CommonModule
BusinessModule
MainModule
```

chứa quá nhiều chức năng không liên quan.

---

### Export tối thiểu cần thiết

Không export toàn bộ Provider.

Chỉ export những gì Module khác cần sử dụng.

---

### Hạn chế Global Module

Global Module giúp giảm Imports.

Tuy nhiên sử dụng quá nhiều sẽ khiến Dependency trở nên khó theo dõi.

---

### Tránh Circular Dependency

Nếu xuất hiện nhiều forwardRef:

```typescript
forwardRef()
```

hãy xem lại thiết kế Module.

---

## 3.20 Common Mistakes

### Quên Export Provider

Sai:

```typescript
@Module({
  providers: [UsersService]
})
export class UsersModule {}
```

Module khác sẽ không thể inject:

```typescript
UsersService
```

---

### Quên Import Module

Sai:

```typescript
@Module({})
export class OrdersModule {}
```

Trong khi đang inject:

```typescript
UsersService
```

NestJS sẽ báo lỗi Dependency Resolution.

---

### Tạo AppModule quá lớn

Không nên:

```typescript
@Module({
  controllers: [
    UserController,
    ProductController,
    OrderController,
    ...
  ]
})
```

Hãy chia nhỏ thành Feature Module.

## 3.21 Ví dụ hoàn chỉnh

Users Module:

```typescript
@Module({
  controllers: [UsersController],
  providers: [UsersService],
  exports: [UsersService]
})
export class UsersModule {}
```

Orders Module:

```typescript
@Module({
  imports: [UsersModule],
  controllers: [OrdersController],
  providers: [OrdersService]
})
export class OrdersModule {}
```

App Module:

```typescript
@Module({
  imports: [
    UsersModule,
    OrdersModule
  ]
})
export class AppModule {}
```

Kiến trúc:

```text
AppModule
├── UsersModule
│   ├── UsersController
│   └── UsersService
│
└── OrdersModule
    ├── OrdersController
    └── OrdersService
```
