# Chương 5. Exception Filters

## 5.1 Mục tiêu chương học

Sau khi hoàn thành chương này, bạn sẽ hiểu:

- Exception là gì trong NestJS.
- NestJS xử lý lỗi như thế nào.
- HttpException hoạt động ra sao.
- Cách tạo Custom Exception.
- Cách xây dựng Exception Filter.
- Cách áp dụng Filter cho Controller hoặc toàn ứng dụng.
- Best Practices khi xử lý lỗi.

---

## 5.2 Exception là gì?

Exception là lỗi xảy ra trong quá trình xử lý Request.

Ví dụ:

```typescript
@Get(':id')
findOne(
  @Param('id')
  id: string
) {

  throw new Error('User not found');

}
```

Khi Exception xuất hiện:

```text
Controller
    ↓
Exception
    ↓
Exception Filter
    ↓
HTTP Response
```

NestJS sẽ bắt lỗi và trả Response tương ứng.

---

## 5.3 Cơ chế xử lý Exception mặc định

Ví dụ:

```typescript
@Get()
findAll() {

  throw new HttpException(
    'Forbidden',
    HttpStatus.FORBIDDEN
  );

}
```

Response:

```json
{
  "statusCode": 403,
  "message": "Forbidden"
}
```

NestJS cung cấp sẵn cơ chế xử lý Exception.

Developer không cần tự tạo try/catch cho mọi Endpoint.

## 5.9 Tại sao cần Exception Filter?

Giả sử toàn hệ thống muốn trả lỗi theo format:

```json
{
  "success": false,
  "message": "User not found",
  "timestamp": "2026-06-19T10:00:00Z"
}
```

Nếu xử lý thủ công:

```typescript
throw new HttpException(...)
```

ở mọi nơi sẽ rất khó quản lý.

Exception Filter giúp xử lý tập trung.

---

## 5.10 Exception Filter là gì?

Exception Filter là thành phần chịu trách nhiệm bắt và xử lý Exception.

Ví dụ:

```text
Controller
    ↓
Exception
    ↓
Exception Filter
    ↓
Response
```

Filter cho phép:

- Ghi log lỗi.
- Chuẩn hóa Response.
- Ẩn thông tin nhạy cảm.
- Thống nhất format lỗi.

## 5.12 Custom Exception Filter đầu tiên

Ví dụ:

```typescript
import {
  ExceptionFilter,
  Catch,
  ArgumentsHost,
  HttpException
} from '@nestjs/common';

@Catch(HttpException)
export class HttpExceptionFilter
  implements ExceptionFilter {

  catch(
    exception: HttpException,
    host: ArgumentsHost
  ) {

    const ctx =
      host.switchToHttp();

    const response =
      ctx.getResponse();

    const request =
      ctx.getRequest();

    response.status(
      exception.getStatus()
    ).json({
      timestamp:
        new Date().toISOString(),
      path: request.url
    });

  }

}
```

---

## 5.13 Decorator @Catch()

Decorator:

```typescript
@Catch()
```

xác định loại Exception cần bắt.

Ví dụ:

```typescript
@Catch(HttpException)
```

chỉ bắt:

```typescript
HttpException
```

---

Bắt nhiều loại:

```typescript
@Catch(
  NotFoundException,
  ForbiddenException
)
```

---

## 5.14 ArgumentsHost

ArgumentsHost cho phép truy cập Context hiện tại.

Ví dụ:

```typescript
host.switchToHttp();
```

Trả về HTTP Context.

---

Lấy Request:

```typescript
const request =
  ctx.getRequest();
```

---

Lấy Response:

```typescript
const response =
  ctx.getResponse();
```

---

## 5.15 Áp dụng Filter cho Method

Ví dụ:

```typescript
@Get(':id')
@UseFilters(
  HttpExceptionFilter
)
findOne() {}
```

Filter chỉ áp dụng cho Endpoint đó.

---

## 5.16 Áp dụng Filter cho Controller

Ví dụ:

```typescript
@UseFilters(
  HttpExceptionFilter
)
@Controller('users')
export class UsersController {}
```

Toàn bộ Endpoint trong Controller sẽ sử dụng Filter.

---

## 5.17 Global Exception Filter

Đăng ký trong:

```typescript
main.ts
```

```typescript
app.useGlobalFilters(
  new HttpExceptionFilter()
);
```

Filter áp dụng cho toàn ứng dụng.

## 5.21 Exception trong Service

Ví dụ:

```typescript
@Injectable()
export class UsersService {

  findOne(id: number) {

    throw new NotFoundException(
      'User not found'
    );

  }

}
```

Controller:

```typescript
@Get(':id')
findOne(
  @Param('id')
  id: string
) {

  return this.usersService.findOne(
    Number(id)
  );

}
```

Exception sẽ tự động được chuyển tới Filter.

## 5.22 Best Practices

### Ném Exception từ Service

Tốt:

```typescript
throw new NotFoundException(
  'User not found'
);
```

trong Service.

Không nên xử lý lỗi thủ công trong Controller.

---

### Chuẩn hóa Error Response

Nên có một format lỗi thống nhất.

Ví dụ:

```json
{
  "success": false,
  "message": "",
  "statusCode": 400
}
```

---

### Sử dụng Built-in Exception

Ưu tiên:

```typescript
NotFoundException
BadRequestException
ConflictException
```

thay vì:

```typescript
HttpException(...)
```

trong các trường hợp phổ biến.

---

### Sử dụng Global Filter

Các hệ thống lớn thường sử dụng:

```typescript
app.useGlobalFilters(...)
```

để quản lý lỗi tập trung.

---

## 5.23 Common Mistakes

### Trả lỗi bằng return

Sai:

```typescript
return {
  error: true
};
```

Nên:

```typescript
throw new BadRequestException();
```

---

### Sử dụng Error thay vì HttpException

Sai:

```typescript
throw new Error(
  'User not found'
);
```

Nên:

```typescript
throw new NotFoundException(
  'User not found'
);
```

---

### Tạo nhiều format lỗi khác nhau

Sai:

```json
{
  "error": true
}
```

và

```json
{
  "status": "failed"
}
```

trong cùng hệ thống.

---

### Xử lý lỗi ở mọi Controller

Không nên:

```typescript
try {
}
catch {
}
```

lặp lại ở mọi Endpoint.

Hãy sử dụng Filter.
