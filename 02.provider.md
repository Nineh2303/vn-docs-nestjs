Sau khi hoàn thành chương này, bạn sẽ hiểu:

- Provider là gì trong NestJS.
- Vì sao Provider là thành phần cốt lõi của framework.
- Cách tạo và đăng ký Provider.

## 2.2 Provider là gì?

Provider là một thành phần có thể được NestJS quản lý và inject vào các thành phần khác.

Trong thực tế, phần lớn Provider xuất hiện dưới dạng Service.

Ví dụ:

```text
Controller
    ↓
Provider
    ↓
Database
```

Controller nhận Request.

Provider xử lý nghiệp vụ.

Ví dụ:

```typescript
@Controller('users')
export class UsersController {

  constructor(
    private readonly usersService: UsersService
  ) {}

}
```

Trong ví dụ trên:

```typescript
UsersService
```

là một Provider.

---

## 2.3 Tại sao cần Provider?

Giả sử Controller tự xử lý mọi thứ:

```typescript
@Controller('users')
export class UsersController {

  @Get()
  findAll() {

    // query database

    // validate

    // business logic

    // logging

    // send email

  }

}
```

Controller sẽ rất khó bảo trì.

Thay vào đó:

```typescript
@Controller('users')
export class UsersController {

  constructor(
    private readonly usersService: UsersService
  ) {}

  @Get()
  findAll() {
    return this.usersService.findAll();
  }

}
```

Business Logic được chuyển sang Provider.

```typescript
@Injectable()
export class UsersService {

  findAll() {
    return [];
  }

}
```

Kiến trúc trở nên rõ ràng hơn.

---

## 2.4 Decorator @Injectable()

Để một class trở thành Provider, cần sử dụng:

```typescript
@Injectable()
```

Ví dụ:

```typescript
import { Injectable } from '@nestjs/common';

@Injectable()
export class UsersService {

  findAll() {
    return [];
  }

}
```

Decorator này cho phép NestJS:

- Quản lý vòng đời đối tượng.
- Đăng ký vào Dependency Injection Container.
- Inject vào các thành phần khác.

---

## 2.5 Tạo Provider bằng CLI

Nest CLI có thể tự động tạo Service.

```bash
nest generate service users
```

Hoặc:

```bash
nest g s users
```

Kết quả:

```text
src/
└── users/
    ├── users.service.ts
    └── users.service.spec.ts
```

Nội dung:

```typescript
@Injectable()
export class UsersService {}
```

---

## 2.6 Đăng ký Provider trong Module

Provider phải được khai báo trong Module.

Ví dụ:

```typescript
@Module({
  providers: [UsersService]
})
export class UsersModule {}
```

NestJS sẽ thêm Provider này vào Dependency Injection Container.

Nếu không khai báo:

```typescript
providers: []
```

NestJS sẽ không thể inject Provider.

---

## 2.7 Sử dụng Provider trong Controller

Ví dụ:

```typescript
@Injectable()
export class UsersService {

  findAll() {
    return [
      'John',
      'Jane'
    ];
  }

}
```

Controller:

```typescript
@Controller('users')
export class UsersController {

  constructor(
    private readonly usersService: UsersService
  ) {}

  @Get()
  findAll() {
    return this.usersService.findAll();
  }

}
```

Request:

```http
GET /users
```

Response:

```json
[
  "John",
  "Jane"
]
```

---

## 2.8 Constructor Injection

NestJS sử dụng Constructor Injection.

Ví dụ:

```typescript
constructor(
  private readonly usersService: UsersService
) {}
```

Khi khởi tạo Controller:

```typescript
UsersController
```

NestJS sẽ:

1. Tìm UsersService.
2. Tạo instance nếu cần.
3. Truyền vào constructor.

Developer không cần:

```typescript
new UsersService()
```

---

## 2.9 Dependency Injection

Dependency Injection là cơ chế cung cấp dependency từ bên ngoài thay vì tự tạo thủ công.

Không nên:

```typescript
const usersService = new UsersService();
```

Nên:

```typescript
constructor(
  private readonly usersService: UsersService
) {}
```

Lợi ích:

- Dễ mở rộng.
- Dễ bảo trì.
- Dễ testing.
- Giảm coupling.

---

## 2.10 Provider Scope

Mặc định:

```typescript
Scope.DEFAULT
```

Provider chỉ được tạo một lần trong suốt vòng đời ứng dụng.

```text
Application
    ↓
One Instance
```

---

### Singleton Scope

Mặc định của NestJS.

```typescript
@Injectable()
export class UsersService {}
```

Chỉ tồn tại một instance.

---

### Request Scope

```typescript
@Injectable({
  scope: Scope.REQUEST
})
```

Mỗi Request tạo một instance mới.

```text
Request A
    ↓
Instance A

Request B
    ↓
Instance B
```

---

### Transient Scope

```typescript
@Injectable({
  scope: Scope.TRANSIENT
})
```

Mỗi lần inject sẽ tạo instance mới.

---

## 2.11 Custom Providers

Ngoài class Provider, NestJS còn hỗ trợ:

- useClass
- useValue
- useFactory
- useExisting

Ví dụ:

```typescript
{
  provide: 'APP_NAME',
  useValue: 'NestJS Course'
}
```

Inject:

```typescript
@Inject('APP_NAME')
private readonly appName: string
```

---

## 2.12 Factory Providers

Factory Provider cho phép tạo dependency động.

Ví dụ:

```typescript
{
  provide: 'CONFIG',
  useFactory: () => {
    return {
      port: 3000
    };
  }
}
```

NestJS sẽ gọi factory khi khởi tạo.

---

## 2.13 Async Providers

Ví dụ:

```typescript
{
  provide: 'DATABASE',
  useFactory: async () => {
    return await createConnection();
  }
}
```

Thường được sử dụng cho:

- Database
- Redis
- Message Queue
- External Services

---

## 2.14 Export Provider

Một Provider chỉ khả dụng trong Module hiện tại.

Ví dụ:

```typescript
@Module({
  providers: [UsersService],
  exports: [UsersService]
})
export class UsersModule {}
```

Sau khi export:

```typescript
imports: [UsersModule]
```

Module khác có thể sử dụng UsersService.

---

## 2.15 Provider Lifecycle

Provider có thể tham gia vào vòng đời ứng dụng.

Ví dụ:

```typescript
@Injectable()
export class UsersService
  implements OnModuleInit {

  onModuleInit() {
    console.log('Module initialized');
  }

}
```

NestJS hỗ trợ:

- OnModuleInit
- OnModuleDestroy
- OnApplicationBootstrap
- OnApplicationShutdown

---

## 2.16 Best Practices

### Một Provider cho một Responsibility

Ví dụ:

```text
UsersService
ProductsService
OrdersService
```

Không nên tạo:

```text
AppService
```

chứa toàn bộ logic hệ thống.

---

### Không xử lý HTTP trong Provider

Không nên:

```typescript
res.status(200).json(...)
```

Provider chỉ nên xử lý nghiệp vụ.

---

### Không truy cập Controller từ Provider

Provider không nên phụ thuộc vào Controller.

Dependency luôn đi theo hướng:

```text
Controller
    ↓
Provider
```

---

## 2.17 Common Mistakes

### Quên @Injectable()

Sai:

```typescript
export class UsersService {}
```

NestJS không thể inject.

---

### Quên khai báo trong providers

Sai:

```typescript
@Module({
  providers: []
})
```

Khi inject:

```typescript
UsersService
```

sẽ phát sinh lỗi.

---

### Tạo instance thủ công

Không nên:

```typescript
new UsersService()
```

Hãy để NestJS quản lý dependency.
