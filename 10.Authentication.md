## 10.2 Authentication là gì?

Authentication là quá trình xác minh danh tính người dùng.

Authentication trả lời câu hỏi:

```text
Bạn là ai?
```

Ví dụ:

Người dùng đăng nhập:

```json
{
  "username": "admin",
  "password": "123456"
}
```

Hệ thống kiểm tra:

- Tài khoản tồn tại?
- Mật khẩu đúng?

Nếu hợp lệ:

```text
Authenticated
```

---

## 10.3 Authentication và Authorization

Đây là hai khái niệm thường bị nhầm lẫn.

### Authentication

```text
Bạn là ai?
```

Ví dụ:

```text
Nguyễn Văn A
```

---

### Authorization

```text
Bạn được phép làm gì?
```

Ví dụ:

```text
Admin
```

được phép:

```text
Create User
Delete User
```

---

Authentication luôn xảy ra trước Authorization.

## 10.4 Kiến trúc Authentication trong NestJS

Một hệ thống JWT thường có luồng:

```text
Client
   ↓
Login
   ↓
AuthService
   ↓
JWT Token
   ↓
Client lưu Token
   ↓
Request tiếp theo
   ↓
AuthGuard
   ↓
Controller
```

## 10.5 JWT là gì?

JWT là viết tắt của:

```text
JSON Web Token
```

JWT là một chuỗi ký tự chứa thông tin người dùng.

Ví dụ:

```text
eyJhbGciOiJIUzI1Ni...
```

JWT thường được gửi qua:

```http
Authorization: Bearer TOKEN
```

---

## 10.6 Cấu trúc JWT

JWT gồm 3 phần:

```text
HEADER
.
PAYLOAD
.
SIGNATURE
```

Ví dụ:

```text
xxxxx.yyyyy.zzzzz
```

---

### Header

```json
{
  "alg": "HS256",
  "typ": "JWT"
}
```

---

### Payload

```json
{
  "sub": 1,
  "username": "admin"
}
```

---

### Signature

Được tạo bằng Secret Key.

Mục đích:

```text
Chống giả mạo Token
```

## 10.11 Validate User

Ví dụ:

```typescript
async validateUser(
  username: string,
  password: string
) {

  const user =
    await this.usersService
      .findByUsername(
        username
      );

  if (!user) {
    return null;
  }

  return user;

}
```

---

Trong thực tế cần kiểm tra mật khẩu bằng:

```typescript
bcrypt.compare()
```

## 10.12 Hash Password

Không bao giờ lưu:

```text
123456
```

trong Database.

Sai:

```json
{
  "password": "123456"
}
```

---

Đúng:

```json
{
  "password":
  "$2b$10$..."
}
```

---

Hash:

```typescript
const hash =
  await bcrypt.hash(
    password,
    10
  );
```

---

## 10.13 So sánh Password

Ví dụ:

```typescript
const matched =
  await bcrypt.compare(
    password,
    user.password
  );
```

Kết quả:

```typescript
true
```

hoặc

```typescript
false
```

## 10.19 JwtStrategy

Ví dụ:

```typescript
@Injectable()
export class JwtStrategy
  extends PassportStrategy(
    Strategy
  ) {

  constructor() {

    super({
      jwtFromRequest:
        ExtractJwt
          .fromAuthHeaderAsBearerToken(),
      ignoreExpiration: false,
      secretOrKey: 'secret-key'
    });

  }

}
```

## 10.20 validate()

Passport sẽ gọi:

```typescript
validate()
```

sau khi giải mã JWT.

Ví dụ:

```typescript
async validate(
  payload: any
) {

  return {
    userId: payload.sub,
    username:
      payload.username
  };

}
```

---

Request sẽ có:

```typescript
request.user
```

## 10.25 Refresh Token

Dùng để sinh Access Token mới.

Ví dụ:

```text
Access Token
    ↓
Hết hạn
    ↓
Refresh Token
    ↓
Token mới
```

---

Thời gian sống:

```text
7 ngày
30 ngày
90 ngày
```

---

## 10.26 Luồng Refresh Token

```text
Login
   ↓
Access Token
Refresh Token
   ↓
Access hết hạn
   ↓
/refresh
   ↓
Access mới
```

## 10.29 Best Practices

### Không lưu Password dạng Plain Text

Sai:

```json
{
  "password": "123456"
}
```

---

### Luôn Hash Password

```typescript
bcrypt.hash()
```

---

### JWT Secret trong Environment

Không hard-code:

```typescript
secret: '123456'
```

---

### Access Token ngắn hạn

Ví dụ:

```text
15 phút
```

---

### Refresh Token dài hạn

Ví dụ:

```text
30 ngày
```

---

### Dùng HTTPS

JWT không mã hóa dữ liệu.

HTTPS là bắt buộc trong môi trường Production.

## 10.30 Common Mistakes

### Lưu Password thuần

Sai:

```typescript
password: '123456'
```

---

### Đưa quá nhiều dữ liệu vào JWT

Sai:

```json
{
  "id": 1,
  "email": "...",
  "address": "...",
  "phone": "...",
  "avatar": "...",
  "roles": [...]
}
```

JWT nên gọn nhẹ.

---

### Token sống quá lâu

Sai:

```text
365 ngày
```

---

### Không kiểm tra Refresh Token

Có thể dẫn đến chiếm quyền truy cập.
