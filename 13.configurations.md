## 13.2 Configuration là gì?

Configuration là tập hợp các giá trị dùng để điều khiển hành vi của ứng dụng mà không cần sửa source code.

Ví dụ:

```text
PORT
DATABASE_URL
JWT_SECRET
REDIS_HOST
SMTP_SERVER
```

---

Ví dụ:

```typescript
const port = 3000;
```

Hard-code như trên là không tốt.

Nên:

```env
PORT=3000
```

---

## 13.3 Tại sao Configuration quan trọng?

Một ứng dụng thường chạy ở nhiều môi trường:

```text
Local
Development
Staging
Production
```

---

Ví dụ:

Local:

```env
DB_HOST=localhost
```

---

Production:

```env
DB_HOST=10.10.10.100
```

---

Nếu hard-code:

```typescript
const dbHost = 'localhost';
```

sẽ rất khó triển khai.

---

## 13.4 Environment Variables

Environment Variable là các biến được hệ điều hành hoặc môi trường runtime cung cấp.

Ví dụ:

```env
PORT=3000

JWT_SECRET=my-secret

DB_HOST=localhost
```

---

NodeJS có thể đọc:

```typescript
process.env.PORT
```

---

Ví dụ:

```typescript
console.log(
  process.env.PORT
);
```

---

## 13.9 ConfigService

Inject:

```typescript
constructor(
  private configService:
    ConfigService
) {}
```

---

Đọc giá trị:

```typescript
const secret =
  this.configService.get(
    'JWT_SECRET'
  );
```

---

## 13.10 Ví dụ sử dụng ConfigService

```typescript
@Injectable()
export class AuthService {

  constructor(
    private configService:
      ConfigService
  ) {}

  getSecret() {

    return this.configService.get(
      'JWT_SECRET'
    );

  }

}
```

## 13.11 ConfigModule Global

Thông thường:

```typescript
 ConfigModule

```

---

## 13.12 Sử dụng Config trong JwtModule

Sai:

```typescript
JwtModule.register({
  secret: 'secret'
})
```

---

Đúng:

```typescript
JwtModule.registerAsync({

  inject: [ConfigService],

  useFactory: (
    configService:
      ConfigService
  ) => ({

    secret:
      configService.get(
        'JWT_SECRET'
      )

  })

})
```

---

## 13.22 Joi Validation

Cài đặt:

```bash
npm install joi
```

---

Ví dụ:

```typescript
import * as Joi
from 'joi';
```

---

Schema:

```typescript
validationSchema:
Joi.object({

  PORT:
    Joi.number()
      .required(),

  JWT_SECRET:
    Joi.string()
      .required()

})
```

---

## 13.23 Validation đầy đủ

```typescript
ConfigModule.forRoot({

  validationSchema:

    Joi.object({

      PORT:
        Joi.number()
          .required(),

      JWT_SECRET:
        Joi.string()
          .required()

    })

})
```

---

Nếu thiếu:

```env
JWT_SECRET=
```

---

Ứng dụng sẽ:

```text
Fail Fast
```

ngay khi khởi động.

## 13.24 Typed Configuration

Thông thường:

```typescript
configService.get(
  'PORT'
)
```

trả về:

```typescript
any
```

---

Có thể chỉ định kiểu:

```typescript
configService.get<number>(
  'PORT'
)
```

---

Ví dụ:

```typescript
const port =
  this.configService.get<number>(
    'PORT'
  );
```

## 13.29 Common Mistakes

### Hard-code cấu hình

Sai:

```typescript
const PORT = 3000;
```

---

### Commit file .env

Sai:

```text
git add .env
```

---

Nên:

```gitignore
.env
```

---

### Không validate cấu hình

Dẫn tới lỗi Production khó phát hiện.

---

### Dùng process.env khắp codebase

Sai:

```typescript
process.env.DB_HOST
process.env.DB_PORT
process.env.JWT_SECRET
```

---

Nên tập trung qua:

```typescript
ConfigService
```

---

### Lưu Secret trong Source Code

Sai:

```typescript
secret:
'my-secret'
```

---
