## 11.2 Authorization là gì?

Authorization là quá trình xác định người dùng được phép thực hiện hành động nào.

Authorization trả lời câu hỏi:

```text
Bạn được phép làm gì?
```

Ví dụ:

Người dùng đã đăng nhập:

```json
{
  "id": 1,
  "username": "john"
}
```

Authentication thành công.

Nhưng khi thực hiện:

```http
DELETE /users/10
```

Hệ thống cần kiểm tra:

```text
John có được xóa User không?
```

Đây chính là Authorization.

---

## 11.3 Authentication vs Authorization

### Authentication

Xác định danh tính.

```text
Bạn là ai?
```

Ví dụ:

```text
John
```

---

### Authorization

Xác định quyền.

```text
Bạn được phép làm gì?
```

Ví dụ:

```text
Create User
Delete User
Update User
```

---

Luôn nhớ:

```text
Authentication trước
Authorization sau
```

  SetMetadata

## 11.17 RolesGuard hoàn chỉnh

```typescript
@Injectable()
export class RolesGuard
  implements CanActivate {

  constructor(
    private reflector:
      Reflector
  ) {}

  canActivate(
    context:
      ExecutionContext
  ): boolean {

    const roles =
      this.reflector.get(
        'roles',
        context.getHandler()
      );

    if (!roles) {
      return true;
    }

    const request =
      context
        .switchToHttp()
        .getRequest();

    return roles.includes(
      request.user.role
    );

  }

}
```

## 11.30 Common Mistakes

### Kiểm tra quyền trong Controller

Sai:

```typescript
if (
  user.role
  !== 'admin'
)
```

---

### Hard-code Role

Sai:

```typescript
if (
 role === 'admin'
)
```

rải rác khắp codebase.

---

### Nhồi toàn bộ logic vào một Guard

Sai:

```text
Authentication
Authorization
Permission
Ownership
```

trong cùng một class.

---

### Không dùng Metadata

Khiến hệ thống khó mở rộng.
