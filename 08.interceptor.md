## 8.2 Interceptor là gì?

Interceptor là thành phần cho phép can thiệp vào quá trình xử lý Request và Response.

Khác với Middleware chỉ xử lý trước Controller, Interceptor có thể:

- Thực thi trước Controller.
- Thực thi sau Controller.
- Thay đổi dữ liệu trả về.
- Bắt và xử lý luồng dữ liệu.

Ví dụ:

```text
Client
   ↓
Interceptor (Before)
   ↓
Controller
   ↓
Provider
   ↓
Interceptor (After)
   ↓
Response
```

---

## 8.3 Vị trí của Interceptor trong Request Lifecycle

Một Request hoàn chỉnh:

```text
Client
   ↓
Middleware
   ↓
Guards
   ↓
Interceptors (Before)
   ↓
Pipes
   ↓
Controller
   ↓
Provider
   ↓
Interceptors (After)
   ↓
Response
```

Interceptor là thành phần duy nhất có khả năng hoạt động ở cả hai chiều.

## 8.6 NestInterceptor Interface

Mọi Interceptor đều triển khai:

```typescript
NestInterceptor
```

Ví dụ:

```typescript
import {
  Injectable,
  NestInterceptor,
  ExecutionContext,
  CallHandler
} from '@nestjs/common';

@Injectable()
export class LoggingInterceptor
  implements NestInterceptor {

  intercept(
    context: ExecutionContext,
    next: CallHandler
  ) {

    return next.handle();

  }

}
```

## 8.7 Phương thức intercept()

Đây là phương thức quan trọng nhất.

```typescript

  next: CallHandler
)
```

Hai thành phần chính:

| Thành phần | Vai trò |
|------------|----------|
| ExecutionContext | Thông tin Request hiện tại |
| CallHandler | Tiếp tục luồng xử lý |

## 8.8 CallHandler

```typescript
next.handle()
```

đại diện cho phần xử lý tiếp theo.

Ví dụ:

```typescript
intercept(
  context,
  next
) {

  console.log('Before');

  return next.handle();

}
```

Luồng:

```text
Before
   ↓
Controller
```

## 8.9 Interceptor trước Controller

Ví dụ:

```typescript
intercept(
  context,
  next
) {

  console.log(
    'Before Controller'
  );

  return next.handle();

}
```

Kết quả:

```text
Before Controller
Controller
```

## 8.10 Interceptor sau Controller

Interceptor sử dụng RxJS để xử lý dữ liệu trả về.

Ví dụ:

```typescript
import { tap } from 'rxjs/operators';

intercept(
  context,
  next
) {

  return next.handle().pipe(
    tap(() => {
      console.log(
        'After Controller'
      );
    })
  );

}
```

Luồng:

```text
Controller
   ↓
After Controller
```

## 8.12 Đo thời gian xử lý Request

Ví dụ:

```typescript
intercept(
  context,
  next
) {

  const now = Date.now();

  return next.handle().pipe(
    tap(() => {

      console.log(
        Date.now() - now
      );

    })
  );

}
```

Kết quả:

```text
35 ms
120 ms
15 ms
```

## 8.13 Response Mapping

Controller:

```typescript
@Get()
findAll() {

  return {
    id: 1,
    name: 'John'
  };

}
```

Interceptor:

```typescript
intercept(
  context,
  next
) {

  return next.handle().pipe(
    map(data => ({
      success: true,
      data
    }))
  );

}
```

Response:

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "John"
  }
}
```

## 8.16 Exception Mapping

Interceptor có thể xử lý lỗi.

Ví dụ:

```typescript
import {
  catchError
} from 'rxjs/operators';
```

```typescript
return next.handle().pipe(
  catchError(error => {

    throw error;

  })
);
```

Tuy nhiên xử lý lỗi tập trung thường nên dùng Exception Filter.

## 8.17 Timeout Interceptor

Ví dụ:

```typescript

return next.handle().pipe(
  timeout(5000)
);
```


## 8.23 APP_INTERCEPTOR

Một cách phổ biến:

```typescript

  providers: [
    {
      provide: APP_INTERCEPTOR,
      useClass: LoggingInterceptor

```

Ưu điểm:

- Hỗ trợ Dependency Injection.
- Có thể inject Service.

## 8.26 So sánh Middleware và Interceptor

| Middleware | Interceptor |
|------------|-------------|
| Chỉ xử lý trước Controller | Trước và sau Controller |
| Không xử lý Response | Có thể xử lý Response |
| Logging Request | Logging Request và Response |
| Không thay đổi dữ liệu trả về | Có thể thay đổi Response |

---

## 8.27 So sánh Guard và Interceptor

| Guard | Interceptor |
|---------|------------|
| Quyết định quyền truy cập | Xử lý luồng dữ liệu |
| Authentication | Response Mapping |
| Authorization | Logging |
| Role Checking | Caching |

## 8.29 Common Mistakes

### Dùng Interceptor để Authorization

Sai:

```typescript
if (role !== 'admin')
```

Nên sử dụng Guard.

---

### Dùng Interceptor để Validation

Sai:

```typescript
if (!email)
```

Nên sử dụng Pipe.

---

### Quá nhiều logic trong một Interceptor

Sai:

```typescript
Logging
Caching
Validation
Authorization
```

trong cùng một class.

---

### Không sử dụng RxJS Operators

Interceptor sẽ mất đi phần lớn sức mạnh nếu chỉ gọi:

```typescript
next.handle();
```

mà không tận dụng:

```typescript
map()
tap()
catchError()
```
