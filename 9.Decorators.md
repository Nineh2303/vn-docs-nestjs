# Chương 9. Custom Decorators

## 9.1 Mục tiêu chương học

Sau khi hoàn thành chương này, bạn sẽ hiểu:

- Decorator là gì trong NestJS.
- Vai trò của Decorator trong kiến trúc NestJS.
- Metadata hoạt động như thế nào.
- Cách tạo Custom Decorator.
- Cách truyền dữ liệu thông qua Decorator.
- Cách kết hợp Decorator với Guard.
- Cách xây dựng các Decorator thường gặp trong dự án thực tế.

---

## 9.2 Decorator là gì?

Decorator là một cú pháp đặc biệt của TypeScript cho phép gắn Metadata vào:

- Class
- Method
- Property
- Parameter

Ví dụ:

```typescript
@Controller('users')
export class UsersController {}
```

Ở đây:

```typescript
@Controller()
```

chính là một Decorator.

---

---

## 9.4 Metadata là gì?

Metadata là dữ liệu mô tả bổ sung cho một đối tượng.

Ví dụ:

```typescript
@Controller('users')
```

NestJS sẽ lưu:

```typescript
{
  path: 'users'
}
```

như một Metadata.

Sau đó Framework đọc Metadata để xây dựng Routing.

---

## 9.5 Tại sao cần Custom Decorator?

Giả sử:

```typescript
request.user
```

được gắn bởi Authentication Guard.

Controller:

```typescript
@Get('profile')
getProfile(
  @Req()
  request
) {

  return request.user;

}
```

Cách này hoạt động nhưng:

- Dài dòng.
- Phụ thuộc Express Request.
- Khó tái sử dụng.

Giải pháp:

```typescript
@Get('profile')
getProfile(
  @CurrentUser()
  user
) {

  return user;

}
```

---

## 9.6 Tạo Decorator đầu tiên

NestJS cung cấp:

```typescript
createParamDecorator()
```

Ví dụ:

```typescript
import {
  createParamDecorator,
  ExecutionContext
} from '@nestjs/common';

export const CurrentUser =
  createParamDecorator(
    (
      data: unknown,
      context: ExecutionContext
    ) => {

      const request =
        context
          .switchToHttp()
          .getRequest();

      return request.user;

    }
  );
```

---

## 9.7 Sử dụng Custom Decorator

Ví dụ:

```typescript
@Get('profile')
getProfile(
  @CurrentUser()
  user
) {

  return user;

}
```

Kết quả:

```typescript
request.user
```

được inject trực tiếp vào Method.

---

## 9.8 ExecutionContext trong Decorator

Custom Decorator cũng sử dụng:

```typescript
ExecutionContext
```

Ví dụ:

```typescript
const request =
  context
    .switchToHttp()
    .getRequest();
```

Điều này cho phép truy cập:

- Request
- Response
- Headers
- User

---

## 9.9 Truyền tham số vào Decorator

Ví dụ:

```typescript
@CurrentUser('email')
```

Decorator:

```typescript
export const CurrentUser =
  createParamDecorator(
    (
      property: string,
      context: ExecutionContext
    ) => {

      const request =
        context
          .switchToHttp()
          .getRequest();

      return request.user[property];

    }
  );
```

---

Controller:

```typescript
@Get()
findMe(
  @CurrentUser('email')
  email: string
) {

  return email;

}
```

## 9.15 SetMetadata

NestJS cung cấp:

```typescript
SetMetadata()
```

để tạo Metadata.

Ví dụ:

```typescript
import {
  SetMetadata
} from '@nestjs/common';
```

---

Decorator:

```typescript
export const Roles =
  (...roles: string[]) =>
    SetMetadata(
      'roles',
      roles
    );
```

---

Sử dụng:

```typescript
@Roles('admin')
```

---

## 9.16 Metadata được lưu ở đâu?

Ví dụ:

```typescript
@Roles('admin')
```

NestJS lưu:

```typescript
{
  roles: ['admin']
}
```

trên Method.

Guard có thể đọc lại thông tin này bằng Reflector.

---

## 9.17 Reflector

Ví dụ:

```typescript
constructor(
  private reflector: Reflector
) {}
```

Đọc Metadata:

```typescript
const roles =
  this.reflector.get(
    'roles',
    context.getHandler()
  );
```

## 9.18 Kết hợp Decorator với Guard

Decorator:

```typescript
@Roles('admin')
```

---

Guard:

```typescript
const roles =
  this.reflector.get(
    'roles',
    context.getHandler()
  );
```

---

Request:

```text
Admin
```

→ Cho phép

```text
User
```

→ Từ chối

---

Đây là mô hình Authorization phổ biến nhất trong NestJS.

## 9.19 Composite Decorator

Một Decorator có thể kết hợp nhiều Decorator khác.

NestJS cung cấp:

```typescript
applyDecorators()
```

Ví dụ:

```typescript
import {
  applyDecorators,
  UseGuards
} from '@nestjs/common';
```

---

```typescript
export function AdminOnly() {

  return applyDecorators(
    Roles('admin'),
    UseGuards(
      AuthGuard,
      RolesGuard
    )
  );

}
```

---

Sử dụng:

```typescript
@AdminOnly()
@Get()
findAll() {}
```

## 9.21 Public Decorator

Ví dụ:

```typescript
export const Public =
  () =>
    SetMetadata(
      'isPublic',
      true
    );
```

---

Controller:

```typescript
@Public()
@Get('login')
login() {}
```

---

Guard:

```typescript
const isPublic =
  this.reflector.get(
    'isPublic',
    context.getHandler()
  );
```

Nếu:

```typescript
true
```

bỏ qua Authentication.

## 9.23 Best Practices

### Tạo Decorator cho logic lặp lại

Ví dụ:

```typescript
@CurrentUser()
```

thay vì:

```typescript
@Req()
request
```

ở mọi Controller.

---

### Đặt tên rõ ràng

Tốt:

```typescript
@CurrentUser()
```

```typescript
@UserId()
```

```typescript
@Public()
```

---

Không nên:

```typescript
@Data()
```

```typescript
@Info()
```

---

### Kết hợp với Guard

Decorator nên mô tả quyền truy cập.

Guard thực hiện kiểm tra.

---

### Giữ Decorator đơn giản

Decorator chủ yếu dùng để:

- Đọc dữ liệu.
- Lưu Metadata.

Không nên chứa Business Logic.

---

## 9.24 Common Mistakes

### Đặt Business Logic trong Decorator

Sai:

```typescript
query database
create order
send email
```

Decorator không nên thực hiện nghiệp vụ.

---

### Không dùng Decorator khi logic lặp lại

Sai:

```typescript
request.user
```

lặp lại trong hàng chục Controller.

---

### Không sử dụng Metadata

Dẫn tới việc Guard khó tái sử dụng.

---

### Decorator quá phức tạp

Một Decorator chỉ nên phục vụ một mục đích cụ thể.
