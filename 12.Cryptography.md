## 12.2 Cryptography là gì?

Cryptography (mật mã học) là lĩnh vực nghiên cứu các kỹ thuật:

- Bảo mật dữ liệu
- Xác thực dữ liệu
- Bảo vệ thông tin nhạy cảm

Ví dụ:

```text
Password
Token
Credit Card
API Key
Secret Key
```

---

## 12.3 Tại sao cần bảo vệ dữ liệu?

Giả sử Database bị rò rỉ:

```sql
users

id | username | password
-------------------------
1  | admin    | 123456
2  | john     | abc123
```

Toàn bộ tài khoản bị lộ ngay lập tức.

---

Một hệ thống đúng phải lưu:

```sql
users

id | username | password
----------------------------------
1  | admin    | $2b$10$...
2  | john     | $2b$10$...
```

Kẻ tấn công không thể đọc trực tiếp mật khẩu.

---

## 12.4 Hashing là gì?

Hashing là quá trình chuyển đổi dữ liệu thành chuỗi có độ dài cố định.

Ví dụ:

```text
123456
```

↓

```text
8d969eef6ecad3c29...
```

---

Đặc điểm:

- Một chiều.
- Không thể giải ngược.
- Dùng để lưu Password.

---

## 12.5 Encryption là gì?

Encryption là quá trình mã hóa dữ liệu.

Ví dụ:

```text
Hello
```

↓

```text
a82ks8d2...
```

---

Người có Key phù hợp có thể giải mã.

```text
Encrypted Data
      ↓
Secret Key
      ↓
Original Data
```

---

## 12.6 Hashing vs Encryption

| Hashing | Encryption |
|----------|------------|
| Một chiều | Hai chiều |
| Không giải mã | Có thể giải mã |
| Dùng cho Password | Dùng cho dữ liệu bí mật |
| Không cần Key | Cần Key |

---

Ví dụ:

### Password

```text
Hashing
```

---

### API Secret

```text
Encryption
```

---

### Credit Card

```text
Encryption
```

---

## 12.7 Tại sao Password không được Encrypt?

Nhiều người mới thường nghĩ:

```text
Encrypt Password
```

là đúng.

Thực tế:

```text
Hash Password
```

mới đúng.

---

Lý do:

Nếu Encrypt:

```text
Database Leak
+
Encryption Key Leak
```

↓

```text
Toàn bộ Password bị lộ
```

---

Hashing không gặp vấn đề này.

---

## 12.8 Salt là gì?

Salt là chuỗi ngẫu nhiên được thêm vào Password trước khi Hash.

Ví dụ:

```text
Password

123456
```

---

Salt:

```text
x9K2aP
```

---

Kết quả:

```text
123456x9K2aP
```

↓

```text
Hash
```

---

## 12.9 Tại sao cần Salt?

Không có Salt:

```text
123456
```

của mọi người dùng đều tạo cùng một Hash.

---

Ví dụ:

```text
User A

123456
```

↓

```text
HASH_A
```

---

```text
User B

123456
```

↓

```text
HASH_A
```

---

Kẻ tấn công dễ dàng dùng Rainbow Table.

---

Salt giải quyết vấn đề này.

---

## 12.10 Rainbow Table Attack

Rainbow Table là tập hợp:

```text
Password
↓
Hash
```

được tạo sẵn.

Ví dụ:

```text
123456
↓
hash1

password
↓
hash2

admin
↓
hash3
```

---

Salt giúp vô hiệu hóa Rainbow Table.

---

## 12.11 bcrypt là gì?

bcrypt là thuật toán Hash Password phổ biến nhất hiện nay.

Ưu điểm:

- Tích hợp Salt.
- Chậm có chủ đích.
- Kháng Brute Force tốt.

---

NestJS thường dùng:

```bash
npm install bcrypt
```

---

## 12.14 So sánh Password

Không bao giờ:

```typescript
password === user.password
```

---

Phải dùng:

```typescript
bcrypt.compare(
  password,
  user.password
)
```

---

Ví dụ:

```typescript
const matched =
  await bcrypt.compare(
    password,
    hash
  );
```

---

Kết quả:

```typescript
true
```

hoặc

```typescript
false
```

## 12.15 AuthService với bcrypt

Ví dụ:

```typescript
async validateUser(
  username: string,
  password: string
) {

  const user =
    await this.usersService
      .findByUsername(
        username
      );

  if (!user) {
    return null;
  }

  const matched =
    await bcrypt.compare(
      password,
      user.password
    );

  if (!matched) {
    return null;
  }

  return user;

}
```

## 12.26 Security Checklist

### Không lưu Plain Password

Sai:

```sql
password = '123456'
```

---

### Luôn Hash Password

```typescript
argon2.hash()
```

hoặc

```typescript
bcrypt.hash()
```

---

### Dùng HTTPS

Bắt buộc trong Production.

---

### Không hard-code Secret

Sai:

```typescript
secret = 'abc'
```

---

### Rotate Secret định kỳ

Đặc biệt với:

```text
JWT Secret
API Secret
Encryption Key
```

---

## 12.27 Common Mistakes

### Lưu Password thuần

Sai:

```typescript
password: '123456'
```

---

### So sánh Password bằng ==

Sai:

```typescript
password === hash
```

---

### Encrypt Password thay vì Hash

Sai:

```typescript
AES(password)
```

---

### Secret Key trong Git

Sai:

```typescript
const SECRET =
'my-secret'
```

commit lên GitHub.

---

### Dùng MD5 hoặc SHA1 cho Password

Sai:

```typescript
md5(password)
```

---

```typescript
sha1(password)
```

---

Các thuật toán này không còn phù hợp để bảo vệ Password.
